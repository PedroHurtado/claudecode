Parte A

MCPs

Conectar Claude Code con herramientas externas, bases de datos y APIs a través de un protocolo unificado.

Model Context Protocol

Qué son los MCPs

Claude Code es el producto. MCP es el protocolo que habla. Claude Code actúa de cliente y se conecta a servidores MCP que exponen capacidades: GitHub, navegador, una BD.

Convierte Claude Code de una herramienta de edición en un agente que actúa sobre tus sistemas.

La analogía

MCP es a las herramientas de IA lo que USB-C a los dispositivos: un conector estándar.

El mismo servidor funciona en Claude Code, Claude Desktop y otros clientes.

Configuración

Los dos tipos de transporte

HTTP

El estándar actual («streamable»). Peticiones HTTP con streaming opcional. Para servidores remotos (un SaaS en la nube).

stdio

Entrada/salida estándar. Para servidores que corren como un proceso local en tu máquina.

Deprecado

Existió un tercer transporte, SSE (Server-Sent Events). Úsalo solo si un servidor todavía lo exige; para todo lo nuevo, HTTP.

El comando

claude mcp add

# Remoto por HTTP (ej. Notion)
claude mcp add --transport http notion \
  https://mcp.notion.com/mcp

# Con Bearer token
claude mcp add --transport http secure \
  https://api.example.com/mcp \
  --header "Authorization: Bearer ..."

# Local por stdio
claude mcp add --transport stdio airtable \
  -- npx -y airtable-mcp-server

Todas las opciones (--transport, --env, --scope) van antes del nombre; el -- separa el comando del servidor stdio.

claude mcp list
claude mcp get <n>
claude mcp remove <n>
/mcp → estado y OAuth

Cómo se comportan

Conocimiento general vs. tus datos

Conocimiento general → respuesta directa

«¿Cómo funcionan las bases de datos de Notion?» se responde sin tocar la tool.

Acceso a tus datos → llamada a la tool

«¿Qué hay en mi base de datos de Proyectos?» sí dispara la herramienta.

Con Tool Search (por defecto), las definiciones de tools se cargan a demanda: añadir servidores tiene impacto mínimo en el contexto. Aun así, no conectes lo que no usas.

Parte B

Hooks

Cambiamos de filosofía: todo lo anterior, Claude lo interpreta. Los hooks son lo contrario: control determinista.

Pedir vs. garantizar

Una regla pide. Un hook garantiza.

Una regla en el CLAUDE.md le pide a Claude que haga algo, y normalmente lo hace.

Un hook —comando de shell, endpoint HTTP o prompt a un LLM— se ejecuta fuera del modelo, disparado por un evento. Si algo debe pasar siempre, sin depender del criterio del modelo, es un hook.

Tipos de handler

Cómo responde un hook

command

Un comando de shell. El más común.

http

Un endpoint que recibe el evento como POST.

prompt · agent

Usan el juicio de un LLM en vez de lógica fija, para decisiones semánticas.

Los tres primeros son deterministas; los dos últimos delegan en el modelo.

Bajo coste de contexto: su configuración vive fuera de la ventana principal.

El hook más popular

Auto-formatear cada fichero editado

{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit|MultiEdit",
      "hooks": [{
        "type": "command",
        "command": "npx prettier --write
          \"$CLAUDE_TOOL_INPUT_FILE_PATH\""
      }]
    }]
  }
}

Cada vez que Claude escribe o edita un fichero, se formatea solo.

Tú no apruebas nada.

Un guardarraíl de seguridad

Bloquear un comando peligroso · el exit 2

"PreToolUse": [{
  "matcher": "Bash",
  "hooks": [{
    "type": "command",
    "command": "echo \"$CLAUDE_TOOL_INPUT\"
      | grep -qE 'rm -rf|DROP TABLE'
      && exit 2 || exit 0"
  }]
}]
exit 2

Bloquea la acción. El stderr se le pasa a Claude como la razón.

exit 0

Permite la acción.

exit 1

Error no bloqueante: la acción sigue. Seguridad → siempre exit 2.

Tres cadencias

Eventos de ciclo de vida

Por sesión
SessionStart

Inyecta contexto fresco (rama, commits). Su salida se añade al contexto.

SessionEnd

Al terminar.

Por turno
UserPromptSubmit

Antes de procesar tu prompt. Puede inyectar o bloquear.

Stop

Puede impedir que el turno termine («los tests fallan»).

Por llamada a tool
PreToolUse

El más potente: aprueba o deniega (exit 2).

PostToolUse

Valida o formatea a posteriori. No puede deshacer.

Conecta con los subagentes

Los guardarraíles cubren también a los subagentes

Tus PreToolUse y PostToolUse se ejecutan para cada tool que use un subagente. Sin eso, un subagente podría saltarse tus guardarraíles. SubagentStop valida cuando uno termina.

El caso que cierra la sesión

Un PreToolUse que bloquea leer ficheros .env (exit 2). Protección garantizada, no «le pedí a Claude que no lo lea». La combinación robusta: un hook + una regla de permiso deny.

El objetivo: sentir la diferencia entre pedir (lo que Claude interpreta) y garantizar (un hook, que se ejecuta pase lo que pase).

Para llevarse a casa

Resumen de la lección

MCP es un protocolo estándar (el «USB-C» de la IA): Claude Code, de cliente, se conecta a servidores externos.

Se añade con claude mcp add (HTTP o stdio), tres ámbitos, y los de proyecto piden aprobación.

Claude llama a la tool para tus datos, no para conocimiento general.

Un hook garantiza un comportamiento fuera del modelo. El exit 2 de un PreToolUse bloquea.

Tres cadencias —sesión, turno, tool— y se disparan también para los subagentes.
